Notes

      In the W3C model, event's will capture from parent to target, then bubble up from target to parent.
      It is possible to tap into these different stages of this movement as if it were a components lifecycle.

      the 3rd boolean parameter of the addEventListener function (useCapture) determines wether or not to run
      the function in the capture (or bubbling) stage.

      if true, the function will be run in the capture stage, else if false or nothing at all, the function will
      be run in the bubbling stage.

      Also, we can use the event.stopPropagation() method to halt any further capture/bubble propagations.
